# 日记功能设计文档

## 概述

日记是「日报的聚合」——用户每天写多条事件生成多份日报，AI 将它们汇总成一篇「平行宇宙日记」。用户也可以跳过日报，直接手写日记。

## 核心逻辑

```
用户写事件 → 生成日报（已实现）
                    ↓
            当天所有日报
                    ↓
        AI 汇总 → 生成日记
                    ↓
            日记页面展示
```

## 日记生成方式

| 方式 | 触发条件 | 说明 |
|------|----------|------|
| AI 自动生成 | 用户手动点击「生成今日日记」 | 汇总当天所有日报，按选定风格生成（Phase 2 先做手动触发，定时自动生成后续迭代） |
| 手写日记 | 用户主动创建 | 用户写内容 → 选择是否 AI 加工 |

## AI 日记风格（三种可选）

### A. 报纸汇编风
- **视觉**：报纸合订本样式，顶部有期号、日期、维度编号
- **内容**：把当天头条串成「今日大事记」，每条事件用 bullet point 列出
- **语气**：严肃新闻标题 + 荒诞内容
- **排版**：信息密度高，节奏快

### B. 散文叙事风
- **视觉**：干净的叙事排版，有标题和署名
- **内容**：AI 用第一人称把一天的事情串成连贯故事，有起承转合
- **语气**：温暖、有故事感
- **排版**：段落式，沉浸感强

### C. 荒诞编年史风（推荐默认）
- **视觉**：「CLASSIFIED · 绝密」标识，档案编号，边框装饰
- **内容**：用严肃的历史档案语气记录「平行宇宙重大事件」
- **语气**：官方、正式、离谱
- **排版**：章节式，每件事一个「重大事件」标题

## 手写日记 AI 加工方式（四种可选）

| 加工方式 | 说明 |
|----------|------|
| 语气转换 | 用户写大白话，AI 转换成选定风格的语气 |
| 荒诞润色 | 保留用户原意，加入平行宇宙的离谱元素 |
| 两者结合 | 先转换语气，再加入荒诞元素 |
| 不加工 | 保留用户原文，不做任何 AI 处理 |

## 风格预设

- 在 `/settings` 页面选择默认 AI 日记风格和手写加工方式
- 生成时直接使用预设，无需每次选择
- 设置页提供预览功能，让用户感受不同风格的效果

## 页面设计

### 路由
- `/diary/[date]` — 日记页面，date 格式为 YYYY-MM-DD

### 布局（左右分栏）

```
┌─────────────────────────────────────────────────┐
│  导航栏：今日 | 日记(高亮) | 周报 | 编年史        │
├──────────────┬──────────────────────────────────┤
│              │                                    │
│  今日事件    │      日记内容展示区                  │
│  (左侧栏)    │      (右侧主区域)                   │
│              │                                    │
│  · 08:30     │   [根据风格渲染不同排版]              │
│  · 12:15     │                                    │
│  · 18:45     │                                    │
│              │                                    │
│  [生成日记]   │                                    │
│  [手动写日记] │                                    │
│              │                                    │
├──────────────┴──────────────────────────────────┤
│  操作栏：重新生成 | 导出长图 | 分享                  │
└─────────────────────────────────────────────────┘
```

### 左侧栏组件

- **标题**：「今日事件」+ 事件数量
- **事件卡片**：每张卡片包含
  - 时间戳（如 08:30）
  - 维度编号（如 维度 7-B）
  - 事件摘要（截取日报头条）
  - 心情标签（极佳/困惑/兴奋等）
  - 天气标签（量子雨/反物质风暴等）
- **生成按钮**：「✨ 生成今日日记」，渐变背景
- **手写入口**：「或 手动写日记」文字链接

### 右侧主区域

#### AI 日记展示
- 根据选定风格渲染不同排版
- 荒诞编年史风格：绝密标识 + 档案编号 + 章节式排版
- 散事风格：标题 + 段落 + 署名
- 报纸汇编风格：期号 + bullet list + 天气板块

#### 手写日记编辑器
- 点击「手动写日记」后，右侧切换为编辑器
- 顶部显示当前风格选择（可临时切换）
- 编辑器：大文本框，支持 Shift+Enter 换行
- 底部：「保留原文」/「AI 加工」按钮
- AI 加工后显示对比视图：原文 vs 加工后

### 操作栏
- **重新生成**：用相同输入重新生成日记
- **导出长图**：使用 html2canvas 导出为图片
- **分享**：生成分享链接（Phase 3+）

## 数据结构

```typescript
// 日记条目
interface DiaryEntry {
  id: string;
  date: string;           // YYYY-MM-DD
  type: "ai" | "handwritten";
  content: string;         // 日记内容（Markdown）
  style?: DiaryStyle;      // AI 日记风格
  processMode?: ProcessMode; // 手写加工方式
  rawContent?: string;     // 手写原文（加工前）
  eventIds: string[];      // 关联的日报 ID
  createdAt: string;       // ISO 时间戳
  updatedAt: string;
}

type DiaryStyle = "newspaper" | "prose" | "chronicle";
type ProcessMode = "tone" | "absurd" | "both" | "none";
```

## 存储方案

使用本地 JSON 文件存储（初期）：

```
data/
  diaries/
    2026-06-03.json    // 每天一个文件，包含当天所有日记
```

## 交互流程

### AI 生成日记
1. 用户点击「生成今日日记」
2. 检查当天是否有日报，无则提示先写事件
3. 收集当天所有日报内容
4. 调用 Claude API，传入风格参数和日报内容
5. 生成日记，保存到本地
6. 渲染展示

### 手写日记
1. 用户点击「手动写日记」
2. 右侧切换为编辑器
3. 用户输入内容
4. 选择加工方式（语气转换/荒诞润色/两者结合/不加工）
5. 如选择加工，调用 Claude API 处理
6. 显示加工结果（可选对比原文）
7. 保存日记

## API 设计

### 生成 AI 日记
```typescript
POST /api/diary/generate
Body: {
  date: string;
  style: DiaryStyle;
  events: Event[];  // 当天所有日报
}
Response: {
  diary: DiaryEntry;
}
```

### 手写日记加工
```typescript
POST /api/diary/process
Body: {
  content: string;
  processMode: ProcessMode;
  style: DiaryStyle;
}
Response: {
  processed: string;
}
```

## 实现阶段

## 视觉规范

沿用现有设计系统：
- 背景色：`#0A0A0F`（void）
- 强调色：`#A78BFA`（quantum）、`#22D3EE`（plasma）、`#F472B6`（nebula-pink）
- 字体：Playfair Display（标题）+ Inter（正文）+ JetBrains Mono（标签）
- 组件风格：暗色卡片 + 发光边框 + 渐变装饰线
- 动效：GSAP 入场 + 滚动 reveal + 悬浮交互
